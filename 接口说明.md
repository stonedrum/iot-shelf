# IoT Shelf 系统 API 接口文档

## 1. 认证模块 (auth.py)

### 1.1 用户登录
- **路径**: `/api/auth/login`
- **方法**: `POST`
- **描述**: 用户登录获取访问令牌
- **请求参数**:
  ```json
  {
    "username": "string",  // 用户名
    "password": "string"   // 密码
  }
  ```
- **响应**: 
  ```json
  {
    "access_token": "string",  // JWT访问令牌
    "token_type": "bearer",   // 令牌类型
    "user": {
      "id": 1,                 // 用户ID
      "username": "admin",    // 用户名
      "phone": "13800138000", // 手机号
      "role": "admin",        // 用户角色 (admin: 超级管理员, general: 普通管理员)
      "city_ids": [1, 2]       // 可访问城市ID列表
    }
  }
  ```
- **权限**: 无

## 2. 用户管理模块 (users.py)

### 2.1 获取当前用户信息
- **路径**: `/api/users/me`
- **方法**: `GET`
- **描述**: 获取当前登录用户的详细信息
- **响应**: 
  ```json
  {
    "id": 1,
    "username": "admin",
    "phone": "13800138000",
    "role": "admin",
    "city_ids": [1, 2]
  }
  ```
- **权限**: 登录用户

### 2.2 获取用户列表
- **路径**: `/api/users`
- **方法**: `GET`
- **描述**: 获取所有用户列表（分页）
- **请求参数**: 
  - `skip`: 跳过条数，默认0
  - `limit`: 返回条数，默认100，最大100
  - `search`: 可选，按用户名或手机号搜索
- **响应**: `List[User]` - 用户列表
- **权限**: 超级管理员 (admin)

### 2.2 获取单个用户详情
- **路径**: `/api/users/{user_id}`
- **方法**: `GET`
- **描述**: 获取指定用户的详细信息
- **响应**: `User` - 用户详情
- **权限**: 超级管理员

### 2.3 创建用户
- **路径**: `/api/users`
- **方法**: `POST`
- **描述**: 创建新用户
- **请求参数**:
  ```json
  {
    "username": "string",      // 用户名
    "password": "string",      // 密码
    "phone": "string",         // 手机号
    "role": "admin|general",   // 用户角色
    "created_by": "string",    // 创建人
    "updated_by": "string"     // 更新人
  }
  ```
- **查询参数**:
  - `city_ids`: 可选，用户可访问的城市ID列表（仅对general角色有效）
- **响应**: `User` - 创建的用户信息
- **权限**: 超级管理员 (admin)

### 2.4 更新用户
- **路径**: `/api/users/{user_id}`
- **方法**: `PUT`
- **描述**: 更新用户信息
- **请求参数**:
  ```json
  {
    "username": "string",      // 可选，用户名
    "password": "string",      // 可选，密码
    "phone": "string",         // 可选，手机号
    "role": "admin|general",   // 可选，用户角色
    "updated_by": "string"     // 更新人
  }
  ```
- **查询参数**:
  - `city_ids`: 可选，用户可访问的城市ID列表（仅对general角色有效）
- **响应**: `User` - 更新后的用户信息
- **权限**: 超级管理员 (admin)

### 2.5 删除用户
- **路径**: `/api/users/{user_id}`
- **方法**: `DELETE`
- **描述**: 删除用户（逻辑删除）
- **响应**: `{"message": "User deleted successfully"}`
- **权限**: 超级管理员

### 2.6 为用户添加城市
- **路径**: `/api/users/{user_id}/cities`
- **方法**: `POST`
- **描述**: 为普通管理员添加可管理城市
- **请求参数**:
  ```json
  {
    "city_ids": [1, 2]  // 城市ID列表
  }
  ```
- **响应**: `{"message": "Cities added successfully"}`
- **权限**: 超级管理员

### 2.7 移除用户城市
- **路径**: `/api/users/{user_id}/cities/{city_id}`
- **方法**: `DELETE`
- **描述**: 移除普通管理员可管理的城市
- **响应**: `{"message": "City removed successfully"}`
- **权限**: 超级管理员

### 2.8 获取当前用户可访问城市
- **路径**: `/api/users/current/cities`
- **方法**: `GET`
- **描述**: 获取当前登录用户可访问的城市列表
- **响应**: `List[City]` - 城市列表
- **权限**: 登录用户
  - 超级管理员 (admin)：返回所有未删除城市
  - 普通管理员 (general)：返回关联城市列表

## 7. 通用说明

### 7.1 认证方式
- 使用JWT Token认证
- 请求头需包含：`Authorization: Bearer <access_token>`

### 7.2 响应格式
- 成功响应：`200 OK` 返回对应数据
- 错误响应：
  ```json
  {
    "detail": "错误信息"
  }
  ```

### 7.3 权限说明
- `admin`: 超级管理员，具有所有操作权限
- `general`: 普通管理员，仅能操作关联城市的数据

### 7.4 分页说明
- 所有列表接口支持分页
- 默认返回100条记录，最大100条
- 分页参数：`skip`（跳过条数）和`limit`（返回条数）

### 7.5 角色说明
- `admin`: 超级管理员，具有所有操作权限
- `general`: 普通管理员，仅能操作关联城市的数据
